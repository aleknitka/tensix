  <name>Task 2: Implement Serial Expert Queue (SEQ) Orchestrator</name>
  <files>src/server/orchestrator.ts</files>
  <action>
    Create `SerialExpertQueue` class that:
    1. Accepts a list of "Experts" (persona, modelId, provider).
    2. Has a `run` method that iterates through experts.
    3. For each expert:
       a. Calls `provider.unload()` for any previously loaded model (if local).
       b. Constructs a prompt using previous conversation context and expert's system prompt.
       c. Calls `provider.generate()` and streams tokens back via a callback.
       d. Appends result to context.
    4. Emits lifecycle events: `expertStarted`, `token`, `expertFinished`, `roundFinished`.
  </action>
