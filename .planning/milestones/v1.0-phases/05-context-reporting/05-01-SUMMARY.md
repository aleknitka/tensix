# Summary: 05-01 Context Management & Summarization

## Objective
Implement automated context management in the SEQ orchestrator via periodic summarization to prevent context window overflow.

## Changes
### src/server/orchestrator.ts
- **Task 1:** Implemented `summarizeMessages` function which:
  - Dynamically identifies the "Blue Hat" persona for the task.
  - Falls back to a hardcoded prompt if no Blue Hat is found.
  - Uses the first available enabled provider if necessary.
  - Returns a concise summary generated by the LLM.
- **Task 2:** Integrated summarization trigger into `runRoundTable`:
  - Defined `MAX_MESSAGES = 10` as the threshold for context reduction.
  - Triggers summarization before a round-table starts if the history exceeds the threshold.
  - Saves the summary back into the database as a "system" message (persistent history).
  - Replaces raw history with the summary for the subsequent agent turns.

## Verification
- **Build Status:** `npm run build` successful.
- **Fix:** Resolved a Next.js prerendering error in `/settings/page` that was blocking the build.

## Success Criteria
- [x] `summarizeMessages` correctly interacts with the LLM (based on logic review).
- [x] `runRoundTable` detects long history and triggers summarization.
