# Requirements: UI Refinements & Dockerization

This milestone focuses on polishing the user interface, improving safety, and providing a robust deployment strategy.

## User Requirements

### 1. Data Safety & Granular Deletion
- **REQ-UI-01**: The system must provide a way to delete all chat history (sessions and messages) without affecting configured models (providers), roles, or personas.
- **REQ-UI-02**: Bulk deletion of models, roles, and personas must be disabled; these items should only be removable individually to prevent accidental data loss.

### 2. Layout & Visibility
- **REQ-UI-03**: The chat window width must be increased to provide a more immersive experience on larger displays (target: `max-w-7xl` or higher).
- **REQ-UI-04**: Improve the contrast ratio for text elements, particularly replacing light grey text on white backgrounds with more legible alternatives.

### 3. Workflow Integrity
- **REQ-UI-05**: The "Refined Prompt" generated by the Refiner agent must be read-only for the user. Users can confirm or skip, but not manually edit the AI-generated refinement output.

### 4. Deployment
- **REQ-OPS-01**: Users should be able to run the entire stack (Frontend + Backend) using a single `docker-compose up` command.
- **REQ-OPS-02**: Configuration must be injectable via environment variables for easy deployment in various environments.

## Technical Requirements

### Backend (Express/Drizzle)
- **T-UI-01**: Modify `/system/reset` endpoint or add a new endpoint `/system/clear-chats` to target only `sessions` and `messages` tables.
- **T-UI-02**: Ensure cascading deletes are correctly handled for session-related data only.

### Frontend (Next.js/Tailwind)
- **T-UI-03**: Update `DebateView.tsx` container width classes.
- **T-UI-04**: Remove edit state and UI controls for the refined prompt in `RefinementView.tsx`.
- **T-UI-05**: Conduct a color audit of `src/components` and update `zinc-400`/`zinc-500` text usage.

### DevOps (Docker)
- **T-OPS-01**: Multi-stage `Dockerfile` for the Next.js app (Standalone mode).
- **T-OPS-02**: `Dockerfile` for the Node.js/Express backend.
- **T-OPS-03**: `docker-compose.yml` with volumes for SQLite persistence.
- **T-OPS-04**: Implement env-config loader to map `.env` variables to the application config.

## Success Criteria
1. User can clear all chats via settings without losing their custom personas or provider configs.
2. Chat window occupies a larger portion of the screen on desktop.
3. Refiner output is locked (no edit button or editable textarea).
4. All text elements pass AA contrast standards.
5. The application builds and runs successfully within Docker containers.
