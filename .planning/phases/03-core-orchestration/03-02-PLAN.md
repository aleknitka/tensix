<task type="auto">
  <name>Task 1: Seed Six Thinking Hats Personas</name>
  <files>src/server/db/seeds/personas.ts</files>
  <action>
    Create a script to populate the `personas` table with the following templates:
    - **White Hat (Facts)**: Focusing on data, facts, information.
    - **Red Hat (Emotions)**: Focusing on feelings, hunches, intuition.
    - **Black Hat (Caution)**: Focusing on risks, problems, weaknesses.
    - **Yellow Hat (Benefits)**: Focusing on benefits, value, optimism.
    - **Green Hat (Creativity)**: Focusing on ideas, alternatives, solutions.
    - **Blue Hat (Process)**: Focusing on control, summary, next steps.

    Set modelId and providerId to `null` by default or use first available.
  </action>
  <verify>
    <automated>sqlite3 sqlite.db "SELECT role FROM personas;" | grep "White Hat"</automated>
  </verify>
  <done>Database contains the six thinking hats as persona records.</done>
</task>

<task type="auto">
  <name>Task 2: Implement Round Table SSE Endpoint</name>
  <files>src/server/index.ts</files>
  <action>
    Implement `POST /sessions/:id/round-table` that:
    1. Accepts a list of persona IDs and optional model overrides.
    2. Uses Hono's `streamSSE` to start a Server-Sent Events stream.
    3. Initializes `SerialExpertQueue` from Task 1.
    4. Streams events:
       - `event: expert_started`, `data: { personaName: 'White Hat' }`
       - `event: token`, `data: { content: '...' }`
       - `event: expert_finished`, `data: { fullContent: '...' }`
    5. After each expert finishes, save the message to the `messages` table.
    6. Finalizes session's `updatedAt`.
  </action>
  <verify>
    <automated>curl -X POST http://localhost:3001/sessions/any-id/round-table -d '{"personas": []}' (should start a stream)</automated>
  </verify>
  <done>Backend can stream sequential multi-agent responses over SSE.</done>
</task>
