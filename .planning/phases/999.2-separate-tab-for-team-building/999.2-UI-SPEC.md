# UI Design Contract: Phase 999.2

This document defines the visual and interaction contract for the refactoring of the session right sidebar into a tabbed layout.

## 1. Visual Standards

### Spacing Scale
- **Base Unit:** 4px
- **Grid:** 8pt (4, 8, 16, 24, 32, 48, 64)
- **Phase Exceptions:**
  - Tab Icons: 14px (lucide-react standard)
  - Sidebar Width: Fixed 400px (existing project standard)

### Typography
| Role | Size | Weight | Line Height | Case |
|------|------|--------|-------------|------|
| Page/Tab Heading | 18px | 700 | 1.2 | Normal |
| Card Title | 14px | 700 | 1.2 | Normal |
| Body Text | 14px | 500 | 1.5 | Normal |
| Meta Label | 10px | 800 | 1.0 | Uppercase |
| Button Label | 12px | 700 | 1.0 | Normal |

- **Font Family:** Geist Sans (sans), Geist Mono (mono) — detected in `layout.tsx`.

### Color Palette
| Token | Light Mode | Dark Mode | Usage |
|-------|------------|-----------|-------|
| Surface | #ffffff | #0a0a0a | Primary background |
| Secondary | #f8fafc (Zinc-50) | #18181b (Zinc-900) | Tab bar, card background |
| Border | #e2e8f0 (Zinc-200) | #27272a (Zinc-800) | Dividers, container outlines |
| Foreground | #09090b (Zinc-950) | #f8fafc (Zinc-50) | Primary text |
| Muted | #71717a (Zinc-500) | #a1a1aa (Zinc-400) | Secondary text, inactive icons |
| Accent | #18181b (Zinc-900) | #f4f4f5 (Zinc-100) | Active tab, primary actions |
| Destructive | #f43f5e (Rose-500) | #f43f5e (Rose-500) | Stop Audit action |

## 2. Component Inventory

### New Components
- **SidebarTabs:** A navigation component at the top of the right sidebar.
  - Icons: `Users` (Team), `FileText` (Docs), `GitBranch` (Branches), `Settings` (Settings), `Wrench` (Tools).
  - State: Active tab management.
- **SequenceConfig:** Extracted from `RoundTableConfig`.
  - Elements: Orchestration Mode buttons, Max Turns slider.
- **ToolList:** A list component for the Tools tab.
  - Elements: List of available tools with name and description.

### Refactored Components
- **RoundTableConfig:** Reduced to focus on Persona Selection and Role Library access.
- **SessionPage (Right Sidebar):** Changed from a vertical stack to a conditional rendering based on active tab.

## 3. Interaction Contract

### Tab Navigation
- **Trigger:** Click on tab icon/label.
- **Effect:** Switch visible content in the sidebar content area.
- **Transition:** None or subtle fade (Tailwind `animate-in fade-in`).
- **Default Tab:** `Team`.

### Configuration
- **Live Save:** All changes to orchestration mode and max turns should be reflected immediately in the session state (consistent with existing pattern).

### Modals
- **Role Library:** Remains a full-screen modal (accessible from the Team tab).
- **Assign Model:** Remains a modal (accessible during persona selection).

## 4. Copywriting Contract

### Primary Labels
- **Tab Titles:** "Team", "Docs", "Branches", "Settings", "Tools"
- **CTAs:**
  - "Start Audit" (Primary)
  - "Stop Audit" (Destructive/Active)
  - "Role Library" (Secondary)
  - "Export JSON" (Meta)
  - "Export Markdown" (Meta)
  - "Generate Audit Report" (Primary)

### Empty States
- **Tools:** "No tools active in this session."
- **Docs:** "No documents uploaded."
- **Branches:** "Single branch session."

### Error Messages
- **Generic:** "Failed to update settings. Please try again."

## 5. Registry Safety Gate
- **Tool:** none.
- **Third-party Registries:** none.

## 6. Implementation Notes
- The "Session Insights" card should be moved to the `Team` tab as a descriptive header.
- The "Export" buttons currently in the header should be moved to the `Settings` tab as per `999.2-CONTEXT.md`.
- The `BranchNavigator`, `DocumentManager`, and `RoundTableConfig` should be updated to remove redundant headers if they are already identified by the Tab.
